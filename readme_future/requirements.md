# WynnTracker Bot コマンドリファクタリング要件定義書

## 概要

WynnTracker BotはWynncraftコミュニティ向けのDiscord Botで、現在8つのメインコマンドを提供しています。本要件定義書は、既存のコマンド実装をより保守性が高く、拡張性のあるアーキテクチャにリファクタリングするための要件を定義します。

## 現在のコマンド一覧

### 1. `/anni` - AI予測Annihilationタイマー（管理者限定）

**ユーザーストーリー:** 管理者として、Annihilationイベントの高精度AI予測システムを使用して、コミュニティメンバーに正確なイベント開始時刻を提供したい。

#### 受入基準

1. WHEN 管理者が `/anni timer` を実行 THEN AI予測に基づく自動更新タイマーが表示される
2. WHEN タイマーが表示される THEN 10秒ごとに自動更新され、信頼度が表示される
3. WHEN 管理者が `/anni predict` を実行 THEN 次回Annihilationの詳細予測が表示される
4. WHEN 管理者が `/anni record` を実行 THEN イベント発生記録が保存される
5. WHEN 管理者が `/anni history` を実行 THEN 過去のイベント履歴が表示される
6. WHEN 管理者が `/anni debug` を実行 THEN 予測システムのデバッグ情報が表示される

### 2. `/guild` - ギルド管理システム

**ユーザーストーリー:** ギルドメンバーとして、週間GXPランキングとレイドランキングを確認して、ギルド内での貢献度を把握したい。

#### 受入基準

1. WHEN ユーザーが `/guild gxp ranking` を実行 THEN 今週のGXPランキングが表示される
2. WHEN ユーザーが `/guild raid ranking` を実行 THEN 今週のレイドランキングが表示される
3. WHEN 管理者が `/guild rank set` を実行 THEN 現在のギルドデータが記録される
4. WHEN 週次リセット時刻になる THEN 自動的にランキングがリセットされる

### 3. `/help` - インタラクティブヘルプシステム

**ユーザーストーリー:** ユーザーとして、ボットの機能を理解するために、分かりやすいヘルプシステムを利用したい。

#### 受入基準

1. WHEN ユーザーが `/help` を実行 THEN ページネーション付きのヘルプメニューが表示される
2. WHEN 管理者が `/help` を実行 THEN 管理者限定コマンドも含むヘルプが表示される
3. WHEN ユーザーがページ切り替えボタンを押す THEN 対応するページが表示される
4. WHEN 1分間操作がない THEN メッセージが自動削除される

### 4. `/lr` - Lootrun情報システム

**ユーザーストーリー:** プレイヤーとして、各キャンプのルートプール情報とMythicアイテムの相場を確認して、効率的なルートランを計画したい。

#### 受入基準

1. WHEN ユーザーが `/lr lootpool` を実行 THEN 各キャンプのルートプールが表示される
2. WHEN ユーザーがページ番号を指定 THEN 過去の週のルートプールが表示される
3. WHEN ユーザーがキャンプを指定 THEN 特定キャンプのみの情報が表示される
4. WHEN ユーザーが `/lr mythranking` を実行 THEN Mythicアイテムの相場ランキングが表示される

### 5. `/raid` - レイド情報システム

**ユーザーストーリー:** プレイヤーとして、今週の各レイドのアスペクト情報とGambit情報を確認して、レイド戦略を立てたい。

#### 受入基準

1. WHEN ユーザーが `/raid aspectpool` を実行 THEN 今週の各レイドのアスペクトが表示される
2. WHEN ユーザーがレアリティを指定 THEN 特定レアリティのアスペクトのみが表示される
3. WHEN ユーザーが言語を指定 THEN 指定言語でアスペクト説明が表示される
4. WHEN アスペクト情報が表示される THEN 現在のGambit情報も合わせて表示される

### 6. `/tm` - Trade Market検索システム

**ユーザーストーリー:** プレイヤーとして、アイテムの最新出品情報と価格履歴を確認して、適正価格での取引を行いたい。

#### 受入基準

1. WHEN ユーザーが `/tm search` を実行 THEN アイテムの最新出品情報が表示される
2. WHEN ユーザーがUnidentifiedオプションを指定 THEN Unidentifiedアイテムのみが表示される
3. WHEN 出品情報が表示される THEN 過去7日間の平均価格も表示される
4. WHEN 30秒以内に再実行 THEN レート制限エラーが表示される

### 7. `/translate` - 高性能翻訳システム

**ユーザーストーリー:** ユーザーとして、多言語コミュニケーションを円滑にするために、高品質な翻訳機能を利用したい。

#### 受入基準

1. WHEN ユーザーが `/translate text` を実行 THEN DeepL APIを使用した高品質翻訳が提供される
2. WHEN 管理者が `/translate auto` を設定 THEN 指定チャンネルで自動翻訳が有効になる
3. WHEN 自動翻訳が有効なチャンネルでメッセージが投稿される THEN 自動的に翻訳される
4. WHEN 原文メッセージにリアクションが追加される THEN 翻訳メッセージにも同期される

### 8. `/wynn` - プレイヤー統計システム

**ユーザーストーリー:** プレイヤーとして、自分や他のプレイヤーの詳細な統計情報を確認して、プレイ状況を把握したい。

#### 受入基準

1. WHEN ユーザーが `/wynn stats` を実行 THEN 指定プレイヤーの統計情報が表示される
2. WHEN 統計情報が表示される THEN プログレスバー付きレベル表示が含まれる
3. WHEN 統計情報が表示される THEN 全5クラスの詳細データが含まれる
4. WHEN 統計情報が表示される THEN ギルド情報とレイド統計が含まれる

## 共通要件

### パフォーマンス要件

1. WHEN APIリクエストが実行される THEN 適切なレート制限が適用される
2. WHEN データが取得される THEN キャッシュシステムが活用される
3. WHEN エラーが発生する THEN 適切なエラーハンドリングが実行される

### セキュリティ要件

1. WHEN 管理者限定コマンドが実行される THEN 適切な権限チェックが行われる
2. WHEN APIキーが使用される THEN 環境変数から安全に取得される
3. WHEN ユーザー入力が処理される THEN 適切なバリデーションが実行される

### 可用性要件

1. WHEN 外部APIが利用不可能 THEN 適切なフォールバック処理が実行される
2. WHEN システムエラーが発生 THEN ユーザーに分かりやすいエラーメッセージが表示される
3. WHEN Bot再起動が発生 THEN 必要なデータが復元される

## リファクタリング対象の課題

### コード重複

1. 各コマンドで類似のエラーハンドリングロジックが重複している
2. レート制限チェックのコードが各コマンドで重複している
3. API呼び出しとキャッシュ処理のパターンが重複している

### 保守性の問題

1. 大きなファイルサイズ（guild.jsは774行）で可読性が低下している
2. 設定値がハードコーディングされている箇所が多い
3. テストが困難な構造になっている

### 拡張性の問題

1. 新しいコマンド追加時の一貫性が保証されていない
2. 共通機能の再利用が困難
3. 設定変更時の影響範囲が不明確

## 成功基準

1. 全ての既存機能が正常に動作する
2. コードの重複が50%以上削減される
3. 新しいコマンド追加時の開発時間が30%短縮される
4. テストカバレッジが80%以上達成される
5. 設定変更時の影響範囲が明確になる
